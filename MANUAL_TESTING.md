# Manual Testing Guide

You can interact with your microservices using PowerShell or any API client like Postman.

## 1. Base URL
All requests should go through the API Gateway at: `http://localhost:5000`

## 2. PowerShell Commands

### A. Register a User
```powershell
$headers = @{ "Content-Type" = "application/json" }
$body = @{
    name = "John Doe"
    email = "john@example.com"
    password = "password123"
} | ConvertTo-Json

Invoke-RestMethod -Method Post -Uri "http://localhost:5000/register" -Headers $headers -Body $body
```

### B. Login
```powershell
$body = @{
    email = "john@example.com"
    password = "password123"
} | ConvertTo-Json

Invoke-RestMethod -Method Post -Uri "http://localhost:5000/login" -Headers $headers -Body $body
```

### C. Create a Product
```powershell
$body = @{
    name = "Gaming Laptop"
    price = 1500.00
} | ConvertTo-Json

Invoke-RestMethod -Method Post -Uri "http://localhost:5000/products" -Headers $headers -Body $body
```

### D. Place an Order
*Note: Replace `user_id` and `product_id` with values returned from previous steps.*
```powershell
$body = @{
    user_id = 1
    product_id = 1
} | ConvertTo-Json

Invoke-RestMethod -Method Post -Uri "http://localhost:5000/orders" -Headers $headers -Body $body
```

### E. Check Notifications
See the notification log generated by the Order Service.
```powershell
Invoke-RestMethod -Method Get -Uri "http://localhost:5000/notifications"
```

## 3. Monitoring Dashboards

- **Prometheus** (Metrics): [http://localhost:9090](http://localhost:9090)
- **Grafana** (Visualizations): [http://localhost:3000](http://localhost:3000)
  - **Username**: `admin`
  - **Password**: `admin`

## 4. View Logs
To see the real-time output of your services:
```bash
# View all logs
docker-compose logs -f

# View specific service logs
docker-compose logs -f notification-service
```
